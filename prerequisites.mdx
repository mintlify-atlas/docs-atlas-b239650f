## System Requirements

Before installing Laravel Permission, ensure your system meets the following requirements:

### PHP Version

<Info>
  **Minimum Required:** PHP 8.4 or higher
</Info>

Laravel Permission requires PHP 8.4 or later. You can check your PHP version by running:

```bash
php --version
```

If you need to upgrade PHP, consult your hosting provider's documentation or use a version manager like [phpenv](https://github.com/phpenv/phpenv) or [Laravel Valet](https://laravel.com/docs/valet).

### Laravel Version

<Info>
  **Supported Versions:** Laravel 12.x and 13.x
</Info>

This package is designed for Laravel 12.0 or higher. Check your Laravel version:

```bash
php artisan --version
```

<Note>
  Using an older version of Laravel? Check the [compatibility table](#version-compatibility) below for the appropriate package version.
</Note>

## Database Requirements

Laravel Permission supports the following database systems:

- **MySQL** 5.7+
- **PostgreSQL** 10.0+
- **SQLite** 3.8.8+
- **SQL Server** 2017+
- **MariaDB** 10.2+

<Tip>
  The package uses standard Laravel migrations, so it works with any database system supported by Laravel.
</Tip>

### Database Configuration

Ensure your `.env` file has the correct database configuration:

```bash .env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=your_database
DB_USERNAME=your_username
DB_PASSWORD=your_password
```

Test your database connection:

```bash
php artisan db:show
```

## Authentication Setup

<Warning>
  Laravel Permission requires an authentication system to be set up in your application.
</Warning>

You need a User model with authentication configured. If you haven't set this up yet:

### Laravel Breeze (Recommended for new projects)

```bash
composer require laravel/breeze --dev
php artisan breeze:install
php artisan migrate
npm install && npm run dev
```

### Laravel Jetstream

```bash
composer require laravel/jetstream
php artisan jetstream:install livewire
php artisan migrate
npm install && npm run dev
```

### Manual Setup

If you prefer a manual setup, ensure you have:
- A `users` table in your database
- A `User` model in `app/Models/User.php`
- Authentication guards configured in `config/auth.php`

## Optional Dependencies

### Laravel Passport (for API Authentication)

If you're using Laravel Passport for API authentication:

```bash
composer require laravel/passport
```

Enable Passport client credentials in the config:

```php config/permission.php
'use_passport_client_credentials' => true,
```

### Laravel Octane (for High Performance)

If you're using Laravel Octane, enable the reset listener:

```php config/permission.php
'register_octane_reset_listener' => true,
```

## Composer Configuration

Ensure your `composer.json` has the correct minimum stability:

```json composer.json
{
    "minimum-stability": "dev",
    "prefer-stable": true
}
```

This ensures you get stable releases while allowing dev dependencies.

<Steps>
  <Step title="Check PHP Version">
    ```bash

    Should show PHP 8.4.0 or higher.
  </Step>

  <Step title="Check Laravel Version">

    ```

  <Step title="Check Database Connection">

  </Step>
</Steps>

## Common Issues

<AccordionGroup>
  <Accordion title="PHP version too old">
    **Error:** `requires php ^8.4`
    
    **Solution:** Upgrade your PHP version to 8.4 or higher. On Ubuntu/Debian:
    
    ```bash
    sudo add-apt-repository ppa:ondrej/php
    sudo apt update
    sudo apt install php8.4
    ```
  </Accordion>

  <Accordion title="Laravel version incompatible">
    **Error:** `laravel/framework[v11.x] require spatie/laravel-permission ^6.0`
    
    **Solution:** Install the correct package version for your Laravel version:
    
    ```bash
    composer require spatie/laravel-permission:^6.0
    ```
  </Accordion>

  <Accordion title="Missing authentication">
    **Error:** User model not found or authentication not configured
    
    **Solution:** Install Laravel Breeze or set up authentication:
    
    ```bash
    composer require laravel/breeze --dev
    php artisan breeze:install
    ```
  </Accordion>

  <Accordion title="Database connection failed">
    **Error:** Database connection errors during installation
    
    **Solution:** Verify your `.env` database credentials and ensure the database exists:
    
    ```bash
    mysql -u root -p
    CREATE DATABASE your_database;
    ```
  </Accordion>
</AccordionGroup>
